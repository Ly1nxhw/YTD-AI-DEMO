# AI 客服助手使用说明书

本文档面向实际使用者和维护者，说明如何启动、配置、使用和维护当前 AI 客服助手项目。

## 1. 项目用途

AI 客服助手是一个基于 `Electron + React + TypeScript` 的本地优先桌面应用，用于跨境电商客服回复生成、话术沉淀、工作区管理和知识库迭代。

它不是单纯的一次性 AI 回复工具，而是围绕“工作区”组织数据的客服知识工作台。每个工作区可以独立保存：

- 话术知识库
- LLM 服务配置
- Prompt 模板
- 记忆文件
- 使用统计
- 学习记录
- 备份和导出文件

适合的使用场景：

- 跨境电商客服团队辅助回复客户消息
- 按品牌、店铺、项目隔离客服知识资产
- 通过真实客服对话持续补充话术库
- 给不同团队交付不含内部业务数据的桌面工具
- 在本地保存业务配置，减少对后端服务的依赖

## 2. 运行环境

开发和本地运行建议环境：

- Windows
- Node.js 18 或更高版本
- npm
- 可用的 OpenAI-compatible 模型服务

项目主要技术栈：

- Electron
- React 19
- TypeScript
- Vite
- Zustand
- Tailwind CSS
- lucide-react

## 3. 开发启动

在项目根目录执行：

```bash
npm install
npm run electron:dev
```

说明：

- `npm install`：安装项目依赖。
- `npm run electron:dev`：同时启动 Vite 前端开发服务和 Electron 桌面应用。

如果只想启动浏览器中的前端页面，可以执行：

```bash
npm run dev
```

注意：完整桌面能力需要 Electron 环境，包括工作区文件读写、剪贴板、全局快捷键、导入导出和 LLM 代理请求。

## 4. 构建打包

构建 Windows 安装包或便携版：

```bash
npm run electron:build
```

当前打包配置会生成 Windows 产物，包括 NSIS 安装包和 portable 便携版。

## 5. 首次使用流程

1. 启动应用。
2. 进入“工作区启动”页面。
3. 选择一种方式：
   - 打开已有工作区
   - 导入工作区 ZIP
   - 新建工作区
4. 如果新建工作区，输入工作区名称并选择保存位置。
5. 进入主工作台后，先配置 LLM。
6. 如果知识库为空，可以使用“初始化工作区”生成首批话术。
7. 粘贴客户消息，开始生成客服回复。

## 6. 工作区说明

工作区是应用的数据边界。一个工作区代表一个独立业务场景，例如一个品牌、店铺、产品线或客户项目。

工作区内通常包含：

```text
workspace.json
prompts.json
providers.json
ui-settings.json
knowledge-base.json
memory/
stats/
backups/
exports/
```

文件和目录含义：

- `workspace.json`：工作区元信息。
- `prompts.json`：Prompt 模板。
- `providers.json`：LLM 服务商配置。
- `ui-settings.json`：界面偏好配置。
- `knowledge-base.json`：话术知识库。
- `memory/`：长期记忆文件。
- `stats/`：使用统计和学习记录。
- `backups/`：备份快照。
- `exports/`：导出文件目录。

程序本身不携带业务数据。真正的话术、模型配置、Prompt 和统计记录都保存在工作区中。

## 7. 配置 LLM

进入应用后，点击右上角设置按钮，切换到“LLM 配置”页。

需要填写：

- API 地址
- API Key
- 模型名称
- 温度
- 最大 Tokens

接口需要兼容 OpenAI `/chat/completions` 协议。

填写完成后点击“测试连接”。连接成功后即可使用回复生成、工作区初始化和从对话学习话术等 AI 功能。

设置页还支持两个模型覆写字段：

- 分流模型：用于 Step 0 / Step 1 分析、意图识别和话术匹配。
- 生成模型：用于 Step 2 最终回复生成。

如果留空，则使用当前 Provider 的默认模型。

## 8. 生成客服回复

主界面中间区域是客服回复生成区。

操作步骤：

1. 在“客户消息”输入框粘贴客户原文。
2. 在顶部语言选择器中选择目标语言，或使用自动识别。
3. 点击“生成回复”。
4. 系统会执行智能分流、语言识别、意图识别、话术库匹配、回复生成和质检。
5. 查看 AI 生成回复和中文对照。
6. 如有变量占位符，使用变量填充面板补全。
7. 必要时手动编辑回复。
8. 点击“确认并复制回复”。
9. 到电商后台、聊天后台或工单后台粘贴发送。

生成结果中可能出现：

- 智能分流结果：判断是否适合自动回复，或需要人工处理。
- 复杂度：客户问题复杂程度。
- 情绪：正面、中性、负面或愤怒。
- 风险等级：低、中、高。
- 匹配话术：从知识库中找到的相关话术。
- AI 生成回复：目标语言回复。
- 中文对照：方便客服审核含义。
- 质检结果：检查语言、敏感词、变量、语气和完整性。

如果系统提示“话术库未覆盖此场景”，说明当前知识库没有命中合适条目。可以将本次回复保存为新话术。

## 9. 保存未覆盖场景为话术

当生成结果显示“话术库未覆盖此场景”时：

1. 点击“将此回复保存为新话术”。
2. 确认或修改话术标题。
3. 确认或修改分类。
4. 点击“保存话术”。

保存后，该话术会进入当前工作区的知识库。之后遇到类似问题，系统会优先匹配这条话术。

## 10. 管理话术库

左侧栏是话术库区域。

支持操作：

- 搜索话术
- 按分类展开话术
- 查看话术正文
- 复制话术
- 编辑话术
- 删除话术
- 新增话术

新增话术步骤：

1. 点击左侧话术库顶部的加号。
2. 选择分类。
3. 填写标题。
4. 填写正文。
5. 填写关键词，多个关键词可用逗号、空格或顿号分隔。
6. 点击保存。

建议话术正文尽量写成可复用模板。订单号、产品名、金额、时间等变化内容建议使用变量占位符。

## 11. 初始化工作区

当新工作区没有知识库时，可以使用“初始化工作区”。

入口：左侧栏点击“初始化工作区”。

使用步骤：

1. 确保已经配置 LLM。
2. 填写品牌名称。
3. 填写产品领域。
4. 填写产品概述。
5. 填写销售渠道，例如 Amazon。
6. 填写履约方式，例如 FBA。
7. 填写目标市场。
8. 填写服务语言。
9. 填写售后策略。
10. 填写禁止承诺项。
11. 选择优先场景。
12. 点击“生成首批话术”。
13. 审核生成结果。
14. 勾选需要保留的话术。
15. 点击“批量写入工作区”。

适合在以下情况使用：

- 新品牌刚开始搭建客服体系。
- 没有现成话术库。
- 新产品上线。
- 新市场扩展。
- 需要快速得到第一版可编辑话术草稿。

## 12. 从对话学习话术

入口：左侧栏点击“从对话学习话术”。

使用步骤：

1. 粘贴一段真实客服对话。
2. 点击“抽取并审阅”。
3. 系统会提取候选话术。
4. 审核每条候选话术的标题、分类、客户表达示例、适用场景、正文和关键词。
5. 查看质量分、风险提示和系统建议。
6. 对每条候选选择：
   - 新建
   - 更新已有
   - 忽略
7. 如选择更新已有，可以查看更新前后差异。
8. 点击“批量写入知识库”。

这个功能适合日常运营后持续补充知识库。建议定期把高质量客服对话导入，让系统沉淀新话术。

## 13. 使用统计

入口：左侧栏点击“使用统计”。

统计数据来自工作区的 `stats/` 目录，主要包括：

- 客服回复生成记录
- 学习话术记录
- 保存话术行为
- 最近学习历史

这些数据可以帮助判断：

- 哪些场景经常出现。
- 哪些问题经常未命中知识库。
- 哪些话术被持续补充。
- 知识库是否在增长。

## 14. 工作区备份与恢复

进入设置页，切换到“工作区”。

可执行操作：

- 导入 ZIP
- 导出 ZIP
- 检查变更
- 立即备份
- 恢复备份
- 重命名工作区
- 切换最近工作区

系统在修改设置、知识库和 memory 时会自动创建备份，最多保留最近 20 份。

建议在以下情况手动备份：

- 大量导入话术前。
- 批量学习对话前。
- 修改 Prompt 前。
- 给别人共享工作区前。
- 版本升级前。

恢复备份会覆盖当前工作区文件，操作前需要确认当前数据是否已经保存或另行备份。

## 15. 导入导出工作区

导出工作区：

1. 打开设置。
2. 进入“工作区”页。
3. 点击“导出 ZIP”。
4. 选择保存位置。

导入工作区：

1. 在启动页选择“导入工作区 ZIP”，或在设置页点击“导入 ZIP”。
2. 选择 ZIP 文件。
3. 系统导入后切换到该工作区。

注意：如果工作区包含 API Key，导出 ZIP 可能包含敏感密钥。共享给他人前建议检查 `providers.json`。

## 16. 桌面辅助功能

顶部工具栏提供：

- 语言选择器：选择回复目标语言。
- 透明度按钮：调整窗口透明度。
- 小窗模式：切换紧凑窗口。
- 置顶按钮：让窗口保持在其他窗口上方。
- 设置按钮：进入配置页。

当前代码会注册全局快捷键：

```text
Ctrl + Shift + Q
```

该快捷键用于唤起窗口并读取剪贴板内容。

## 17. 推荐日常工作流

新项目第一次使用：

1. 新建工作区。
2. 配置 LLM。
3. 测试连接。
4. 使用“初始化工作区”生成第一批话术。
5. 人工审核并保存。
6. 开始处理客户消息。

日常客服使用：

1. 从客服后台复制客户消息。
2. 应用自动读取或手动粘贴。
3. 点击生成回复。
4. 审核中文对照和目标语言回复。
5. 必要时编辑。
6. 点击确认并复制。
7. 粘贴回客服后台。

知识库维护：

1. 遇到未覆盖问题，保存为新话术。
2. 每天或每周整理真实客服对话。
3. 用“从对话学习话术”批量提取。
4. 审核后写入知识库。
5. 定期导出工作区 ZIP 备份。

## 18. 开发者目录说明

核心目录：

```text
electron/
  main.ts        Electron 主进程，本地文件、IPC、备份、导入导出、LLM 代理
  preload.ts     向渲染层暴露 window.electronAPI

src/
  App.tsx        应用主入口和页面切换
  components/    UI 面板
  stores/        Zustand 状态管理
  lib/           LLM、变量、学习校验、知识合并等基础逻辑
  agent/         回复生成 pipeline、上下文、质检、记忆、统计
  types/         类型定义

docs/
  项目方案和使用文档

ai-assets/
  仓库地图、架构说明、约定和项目概览
```

常用命令：

```bash
npm install
npm run electron:dev
npm run electron:build
npx tsc --noEmit
```

说明：

- `npm run electron:dev`：本地开发运行桌面应用。
- `npm run electron:build`：打包 Windows 产物。
- `npx tsc --noEmit`：执行 TypeScript 类型检查。

## 19. 注意事项

- 当前项目更适合人工辅助客服，不是自动发送消息的全自动客服系统。
- 工作区导出可能包含 API Key，共享前需要检查敏感配置。
- 当前仓库没有独立测试脚本，发布前建议至少执行 `npx tsc --noEmit`。
- 修改 Prompt 前建议先创建手动备份。
- 批量学习对话写入知识库前建议先备份当前工作区。
